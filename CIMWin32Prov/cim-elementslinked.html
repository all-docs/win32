CIM_ElementsLinked class

The CIM_ElementsLinked association represents physical elements that are cabled together by a physical link.

[!IMPORTANT] The DMTF (Distributed Management Task Force) CIM (Common Information Model) classes are the parent classes upon which WMI classes are built. WMI currently supports only the CIM 2.x version schemas.

The following syntax is simplified from Managed Object Format (MOF) code and includes all inherited properties. Properties are listed in alphabetic order, not MOF order.

Syntax

[Abstract, UUID("{FAF76B83-798C-11D2-AAD1-006008C78BC7}"), AMENDMENT]
class CIM_ElementsLinked : CIM_Dependency
{
  CIM_PhysicalElement REF Dependent;
  CIM_PhysicalLink    REF Antecedent;
};

Properties

The CIM_ElementsLinked class has these properties.

Antecedent

Data type: CIM_PhysicalLink

Access type: Read-only

Qualifiers: Override ("Antecedent")

A CIM_PhysicalLink that describes the physical link.

Dependent

Data type: CIM_PhysicalElement

Access type: Read-only

Qualifiers: Override ("Dependent")

A CIM_PhysicalElement that describes the physical element that is linked.

Remarks

The CIM_ElementsLinked class is derived from CIM_Dependency.

WMI does not implement this class.
