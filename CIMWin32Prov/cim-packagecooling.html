CIM_PackageCooling class

The CIM_PackageCooling association represents the relationship in which a cooling device is installed in a package, such as a chassis or rack, to assist with the package's cooling.

[!IMPORTANT] The DMTF (Distributed Management Task Force) CIM (Common Information Model) classes are the parent classes upon which WMI classes are built. WMI currently supports only the CIM 2.x version schemas.

The following syntax is simplified from Managed Object Format (MOF) code and includes all inherited properties. Properties are listed in alphabetic order, not MOF order.

Syntax

[Abstract, UUID("{FAF76B8E-798C-11D2-AAD1-006008C78BC7}"), AMENDMENT]
class CIM_PackageCooling : CIM_Dependency
{
  CIM_PhysicalPackage REF Dependent;
  CIM_CoolingDevice   REF Antecedent;
};

Properties

The CIM_PackageCooling class has these properties.

Antecedent

Data type: CIM_CoolingDevice

Access type: Read-only

Qualifiers: Override ("Antecedent")

A CIM_CoolingDevice that describes the cooling device for the package.

Dependent

Data type: CIM_PhysicalPackage

Access type: Read-only

Qualifiers: Override ("Dependent")

A CIM_PhysicalPackage that describes the physical package whose environment is cooled.

Remarks

CIM_PackageCooling is derived from CIM_Dependency.

WMI does not implement this class.
