CIM_DeviceAccessedByFile class

The CIM_DeviceAccessedByFile association class specifies the logical device accessed by using the referenced CIM_DeviceFile class.

[!IMPORTANT] The DMTF (Distributed Management Task Force) CIM (Common Information Model) classes are the parent classes upon which WMI classes are built. WMI currently supports only the CIM 2.x version schemas.

The following syntax is simplified from Managed Object Format (MOF) code and includes all inherited properties. Properties are listed in alphabetic order, not MOF order.

Syntax

[Abstract, UUID("{05D1FFF2-DB36-11d2-85FC-0000F8102E5F}"), AMENDMENT]
class CIM_DeviceAccessedByFile : CIM_Dependency
{
  CIM_LogicalDevice REF Dependent;
  CIM_DeviceFile    REF Antecedent;
};

Properties

The CIM_DeviceAccessedByFile class has these properties.

Antecedent

Data type: CIM_DeviceFile

Access type: Read-only

Qualifiers: Override ("Antecedent")

A CIM_DeviceFile that describes the device file.

Dependent

Data type: CIM_LogicalDevice

Access type: Read-only

Qualifiers: Max (1), Override ("Dependent")

A CIM_LogicalDevice that describes the device that is accessed using the device file.

Remarks

WMI does not implement this class.

The CIM_DeviceAccessedByFile class is derived from CIM_Dependency.
