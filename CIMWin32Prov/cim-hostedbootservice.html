CIM_HostedBootService class

The CIM_HostedBootService class associates a hosting system and a boot service. Since this relationship is subclassed from CIM_HostedService, it inherits the scoping/naming scheme defined for service, where a service defers to its hosting system.

[!IMPORTANT] The DMTF (Distributed Management Task Force) CIM (Common Information Model) classes are the parent classes upon which WMI classes are built. WMI currently supports only the CIM 2.x version schemas.

The following syntax is simplified from Managed Object Format (MOF) code and includes all inherited properties. Properties are listed in alphabetic order, not MOF order.

Syntax

[Abstract, UUID("{6DAE7092-DB36-11d2-85FC-0000F8102E5F}"), AMENDMENT]
class CIM_HostedBootService : CIM_HostedService
{
  CIM_System      REF Antecedent;
  CIM_BootService REF Dependent;
};

Properties

The CIM_HostedBootService class has these properties.

Antecedent

Data type: CIM_System

Access type: Read-only

Qualifiers: Override (Antecedent)

A CIM_System that describes the hosting system.

Dependent

Data type: CIM_BootService

Access type: Read-only

Qualifiers: Override ("Dependent")

A CIM_BootService that describes the boot service hosted on the system.

Remarks

The CIM_HostedBootService class is derived from CIM_HostedService.

WMI does not implement this class.
