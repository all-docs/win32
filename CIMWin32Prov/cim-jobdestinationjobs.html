CIM_JobDestinationJobs class

The CIM_JobDestinationJobs association describes where a job is submitted for processing (that is, to which job destination).

[!IMPORTANT] The DMTF (Distributed Management Task Force) CIM (Common Information Model) classes are the parent classes upon which WMI classes are built. WMI currently supports only the CIM 2.x version schemas.

The following syntax is simplified from Managed Object Format (MOF) code and includes all inherited properties. Properties are listed in alphabetic order, not MOF order.

Syntax

[Abstract, UUID("{8D079BEE-DB36-11d2-85FC-0000F8102E5F}"), AMENDMENT]
class CIM_JobDestinationJobs : CIM_Dependency
{
  CIM_Job            REF Dependent;
  CIM_JobDestination REF Antecedent;
};

Properties

The CIM_JobDestinationJobs class has these properties.

Antecedent

Data type: CIM_JobDestination

Access type: Read-only

Qualifiers: Max (1), Override ("Antecedent")

A CIM_JobDestination describing the job destination, possibly a queue.

Dependent

Data type: CIM_Job

Access type: Read-only

Qualifiers: Override ("Dependent")

A CIM_Job describing the job that is in the job queue/Destination.

Remarks

CIM_JobDestinationJobs is derived from CIM_Dependency.

WMI does not implement this class.
