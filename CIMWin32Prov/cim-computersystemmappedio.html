CIM_ComputerSystemMappedIO class

The CIM_ComputerSystemMappedIO class represents an association between a computer system and its available memory-mapped I/O ports.

[!IMPORTANT] The DMTF (Distributed Management Task Force) CIM (Common Information Model) classes are the parent classes upon which WMI classes are built. WMI currently supports only the CIM 2.x version schemas.

The following syntax is simplified from Managed Object Format (MOF) code and includes all inherited properties. Properties are listed in alphabetic order, not MOF order.

Syntax

[Abstract, UUID("{A2EFC897-E3D3-11d2-8601-0000F8102E5F}"), AMENDMENT]
class CIM_ComputerSystemMappedIO : CIM_ComputerSystemResource
{
  CIM_ComputerSystem REF GroupComponent;
  CIM_MemoryMappedIO REF PartComponent;
};

Properties

The CIM_ComputerSystemMappedIO class has these properties.

GroupComponent

Data type: CIM_ComputerSystem

Access type: Read-only

Qualifiers: Override (GroupComponent)

A CIM_ComputerSystem that describes the computer system mapped to the I/O port.

This property is inherited from CIM_ComputerSystemResource

PartComponent

Data type: CIM_MemoryMappedIO

Access type: Read-only

Qualifiers: Override ("PartComponent"), Weak

A CIM_MemoryMappedIO that describes a memory mapped I/O port of the computer system.

Remarks

The CIM_ComputerSystemMappedIO class is derived from CIM_ComputerSystemResource.

WMI does not implement this class.
