CIM_BIOSLoadedInNV class

The CIM_BIOSLoadedInNV class associates a BIOS element and the non-volatile storage in which it is loaded.

[!IMPORTANT] The DMTF (Distributed Management Task Force) CIM (Common Information Model) classes are the parent classes upon which WMI classes are built. WMI currently supports only the CIM 2.x version schemas.

The following syntax is simplified from Managed Object Format (MOF) code and includes all inherited properties. Properties are listed in alphabetic order, not MOF order.

Syntax

[Abstract, UUID("{524ED194-DB35-11d2-85FC-0000F8102E5F}"), AMENDMENT]
class CIM_BIOSLoadedInNV : CIM_Dependency
{
  CIM_BIOSElement        REF Dependent;
  CIM_NonVolatileStorage REF Antecedent;
  uint64                     EndingAddress;
  uint64                     StartingAddress;
};

Properties

The CIM_BIOSLoadedInNV class has these properties.

Antecedent

Data type: CIM_NonVolatileStorage

Access type: Read-only

Qualifiers: Override ("Antecedent")

A CIM_NonVolatileStorage that describes the non-volatile storage.

Dependent

Data type: CIM_BIOSElement

Access type: Read-only

Qualifiers: Override ("Dependent")

A CIM_BIOSElement that describes the BIOS stored in the non-volatile extent.

EndingAddress

Data type: uint64

Access type: Read-only

Ending address where the BIOS is located in non-volatile storage.

For more information about using uint64 values in scripts, see Scripting in WMI.

StartingAddress

Data type: uint64

Access type: Read-only

Starting address where the BIOS is located in non-volatile storage.

For more information about using uint64 values in scripts, see Scripting in WMI.

Remarks

The CIM_BIOSLoadedInNV class is derived from CIM_Dependency.

WMI does not implement this class.
