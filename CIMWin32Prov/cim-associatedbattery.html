CIM_AssociatedBattery class

The CIM_AssociatedBattery dependency associates a battery with a logical device. Use this association to model individual batteries that make up an uninterruptible power supply (UPS).

[!IMPORTANT] The DMTF (Distributed Management Task Force) CIM (Common Information Model) classes are the parent classes upon which WMI classes are built. WMI currently supports only the CIM 2.x version schemas.

The following syntax is simplified from Managed Object Format (MOF) code and includes all inherited properties. Properties are listed in alphabetic order, not MOF order.

Syntax

[Abstract, UUID("{8502C578-5FBB-11D2-AAC1-006008C78BC7}"), AMENDMENT]
class CIM_AssociatedBattery : CIM_Dependency
{
  CIM_LogicalDevice REF Dependent;
  CIM_Battery       REF Antecedent;
};

Properties

The CIM_AssociatedBattery class has these properties.

Antecedent

Data type: CIM_Battery

Access type: Read-only

Qualifiers: Override ("Antecedent")

A CIM_Battery that describes the battery.

Dependent

Data type: CIM_LogicalDevice

Access type: Read-only

Qualifiers: Override ("Dependent")

A CIM_LogicalDevice that contains the logical device needing or associated with the battery.

Remarks

The CIM_AssociatedBattery class is derived from CIM_Dependency.

WMI does not implement this class. For more information about classes derived from CIM_AssociatedBattery, see Win32 Classes.
