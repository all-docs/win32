Routing Table Manager Instance

An instance is a separate table that the routing table manager uses to maintain routing information about a subset of interfaces. Instances are typically used to enable a physical router to act as a set of virtual routers â€“ one virtual router per logical network.

Currently, the routing table manager supports only one instance (identified as zero, the default). The client can register with other instances, but no virtual router except the default one is recognized or used by the router manager.
