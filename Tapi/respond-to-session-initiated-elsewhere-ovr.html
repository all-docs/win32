Respond to Session Initiated Elsewhere

When a new communications session arrives, TAPI applications that registered an interest in the address and media type will be sent an event notification. Only one application will be offered ownership privileges over the session. This application cannot refuse the session.

The initial call state of an incoming session is offering. While the call is in the offering state, an application can obtain information such as bearer mode and call reason, if the service providers supplied this information and it is available. Some call information may not be available immediately, such as caller ID.

There are six basic responses to an offered session: answer, accept, handoff, drop, forward, and redirect. Depending on the service provider, the full set of these operations may not be available.
