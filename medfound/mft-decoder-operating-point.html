Remarks

The term â€œoperating pointâ€ comes from the AV1 specification, but can apply generally to layering methods of a variety of video codecs. The definition of the layer varies depending on the video format. For video formats that support both spatial and temporal scalability, a given operating point may allow for both spatial and temporal scalability â€“ those features are not necessarily exclusive. Generally, the higher the layer, the more frames are output by the decoder. This is useful when an app knows that it cannot support layers beyond a certain value. For example, higher layers may be too CPU intensive or use a higher resolution than is required by the app. Specifying this value allows the decoder to save resources by not decoding unwanted layers in the bitstream.

There is no upper bound on this value. If the value is higher than the highest coded layer in the bitstream, then all layers are emitted. This attribute should be set on the decoderâ€™s MFT attribute store that can be obtained by invoking IMFTransform::GetAttributes.

By default, this value is 0xffffffff to emit all layers.
