SPFILENOTIFY_COPYERROR message

The SPFILENOTIFY_COPYERROR notification is sent to the callback routine if an error occurs during a file copy operation.

SPFILENOTIFY_COPYERROR
  Param1 = (UINT_PTR) FilePathInfo;
  Param2 = (UINT_PTR) ReturnBuffer;

Parameters

Param1

Pointer to a FILEPATHS structure.

Param2

Pointer to a buffer, of size MAX_PATH characters, that stores new path information specified by the user.

Return value

The callback should return one of the following values.

Return code Description
FILEOP_ABORT Queue processing should be canceled. SetupCommitFileQueue returns zero and GetLastError returns extended error information such as ERROR_CANCELLED (if the user canceled) or ERROR_NOT_ENOUGH_MEMORY.
FILEOP_NEWPATH Retry the copy operation using the path the callback function placed in the buffer pointed to by the Param2 parameter. The callback routine should ensure that the path does not overflow the buffer size of a TCHAR array of MAX_PATH elements.
FILEOP_RETRY The user is attempting the copy operation again.
FILEOP_SKIP The user is skipping the file copy operation.

Requirements

Requirement Value
