LUN Resynchronization

In Windows ServerÂ 2008Â R2 and WindowsÂ 7, VSS requesters can use a hardware shadow copy provider feature called LUN resynchronization (or "LUN resync"). This is a fast-recovery scheme that allows an application administrator to restore data from a shadow copy to the original logical unit number (LUN) or to a new LUN. The shadow copy can be a full clone or a differential shadow copy. In either case, at the end of the resync operation, the destination LUN will have the same contents as the shadow copy LUN. During the resync operation, the array performs a block-level copy from the shadow copy to the destination LUN. For more information, see the following:

  • LUN Resync - A fast recovery scenario in Server 2008 R2
  • "How Shadow Copies Are Used" in Volume Shadow Copy Service
