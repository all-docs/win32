IVMGuestOS::MultipleUserSessionsAllowed property

[Windows Virtual PC is no longer available for use as of Windows 8. Instead, use the Hyper-V WMI provider (V2).]

Determines whether multiple simultaneous user sessions are allowed in the guest operating system.

This property is read-only.

Syntax

HRESULT get_MultipleUserSessionsAllowed(
  [out, retval] VARIANT_BOOL *sessionStatus
);

Property value

VARIANT_TRUE if multiple simultaneous user sessions are allowed in the guest operating system, VARIANT_FALSE otherwise.

Error codes

Name/value Meaning
S_OK 0 The operation was successful.
S_FALSE 1 Remote Desktop Services (formerly known as Terminal Services) is not yet initialized in the guest operating system.
E_POINTER 0x80004003 The sessionStatus parameter is NULL.
VM_E_VM_NOT_RUNNING 0xA0040206 The virtual machine is not running.
VM_E_ADDITIONS_FEATURE_NOT_AVAIL 0xA0040505 Integration components are not installed in this virtual machine.
DISP_E_EXCEPTION 0x80020009 An unexpected error has occurred.

Remarks

The value of this property is not valid unless the TerminalServicesInitialized property is VARIANT_TRUE.

For Windows client operating systems, MultipleUserSessionsAllowed is VARIANT_TRUE if Fast User Switching is supported. For Windows Server operating systems, MultipleUserSessionsAllowed is VARIANT_TRUE if Remote Desktop Services (formerly called Terminal Services) is installed and enabled.
