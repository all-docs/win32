/sal switch

The /sal switch directs MIDL to generate SAL annotations in the generated stub files.

midl /sal

Switch Options

This switch has no parameters.

Remarks

MIDL will mark pointer and array parameters with annotations that reflect the parameter description in the IDL file as enforced by RPC and the NDR marshaling engine. MIDL does not create annotations for parameters in interface methods marked with the [local]attribute unless /sal_local is also present on the command line. To override the MIDL-generated annotation, use the [annotate] attribute.

MIDL-generated annotations are always prefixed with __RPC_, and require use of the rpcsal.h header to translate the annotation into standard SAL annotations.
