Passing null as the Argument of Windows Installer Functions

Windows Installer functions that return data in a user providedâ€“memory location should not be called with null as the value for the pointer. These functions return a string or return data as integer pointers, but return inconsistent values when passing null as the value for the output argument.

Do not pass Null as the value of the output argument for any of the following functions:

MsiGetProperty

MsiRecordGetString

MsiFormatRecord

MsiGetSourcePath

MsiGetTargetPath

MsiGetFeatureState

MsiViewGetError

MsiSummaryInfoGetProperty

MsiEvaluateCondition

MsiGetFeatureCost

MsiGetFeatureState

MsiGetComponentState

MsiGetFeatureCost

MsiGetFeatureValidStates
