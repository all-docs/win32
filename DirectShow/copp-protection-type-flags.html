COPP Protection Type Flags

[The feature associated with this page, DirectShow, is a legacy feature. It has been superseded by MediaPlayer, IMFMediaEngine, and Audio/Video Capture in Media Foundation. Those features have been optimized for Windows 10 and Windows 11. Microsoft strongly recommends that new code use MediaPlayer, IMFMediaEngine and Audio/Video Capture in Media Foundation instead of DirectShow, when possible. Microsoft suggests that existing code that uses the legacy APIs be rewritten to use the new APIs if possible.]

The follow constants are used with Certified Output Protection Protocol (COPP) to specific output protection mechanisms.

Constant/value Description
COPP_ProtectionType_Unknown 0x80000000 Unknown protection mechanism.
COPP_ProtectionType_None 0x00000000 No protection mechanisms.
COPP_ProtectionType_HDCP 0x00000001 High-Bandwidth Digital Content Protection (HDCP).
COPP_ProtectionType_ACP 0x00000002 Analog Copy Protection (ACP).
COPP_ProtectionType_CGMSA 0x00000004 Copy Generation Management System Analog (CGMS-A).
COPP_ProtectionType_Mask 0x80000007 Bitmask to isolate valid flags.
COPP_ProtectionType_Reserved 0x7FFFFFF8 Reserved. Must be zero.

Remarks

Some methods return a single flag from this enumeration type, and some methods return a bitwise OR of one or more flags.

These flags are used in the following COPP queries and commands.

  • Global Protection Level
  • Local Protection Level
  • Protection Type
  • Set Protection Level
