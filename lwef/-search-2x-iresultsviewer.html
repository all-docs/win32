IResultsViewer interface

[!NOTE] Windows Desktop Search 2.x is an obsolete technology that was originally available as an add-in for WindowsÂ XP and Windows Server 2003. On later releases, use the Windows Search API instead.

Exposes methods and properties for the results view.

Members

The IResultsViewer interface inherits from the IUnknown interface. IResultsViewer also has these types of members:

  • Methods
  • Properties

Methods

The IResultsViewer interface has these methods.

Method Description
GoBack Not implemented.
GoForward Not implemented.
Refresh Not implemented.
Stop Not implemented.
Update Applies any query changes and navigates the view to the new set of results.

Properties

The IResultsViewer interface has these properties.

Property Access type Description
Contents Write-only This property tracks the type of the content being displayed in the results view.
DisplayName Read-only Localized display name of the type.
EnumSelectedItems Write-only Not implemented.
FilterType Read/write This property will set or return the name of the preceived type to filter results by.
HeaderStyle Read/write The style of header displayed in the view.
IsUpdateNeeded Write-only This returns TRUE if the views query has been modified and needs updating.
ItemStore Read/write This property will set or return the name of the store to filter results by.
PreviewStyle Read/write Controls the preview pane's display mode.
PropertyFilters Write-only When calling the property filter collection it will return the following:
QueryText Read/write Gets or sets the current query text.
ResultsStyle Write-only Not implemented.
SortOrderProperty Read/write This property will set or return the order of columns to sort results by.
SortProperty Read/write This property will set or return the IndexColumn of the property to sort results by.

Remarks

These methods and properties are used to manipulate the information viewed.

Requirement Value
