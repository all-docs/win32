Configuring Library Applications

Because COM+ library applications run in the client's process, the configurable elements for library applications are considerably different than for server applications. You cannot configure any aspect of the application that is determined by the hosting process.

At the application level, several settings are either limited or unavailable for library applications. These constraints are described in the following table:

Attribute Description
Security level You must choose component-level access checks. The library application cannot influence process-level access checks. See Setting a Security Level for Access Checks.
Enabling or disabling authentication You can indicate whether the library application will participate in authentication performed by the host process. See Enabling Authentication for a Library Application.
Impersonation level Unavailable. The setting of the host process is used.
Security identity Unavailable. The application runs under the identity of the host process.
Server process shutdown Unavailable.
Enable 3-GB support Unavailable.
Launch in debugger Unavailable.
Enable CRM Unavailable.
Queuing Unavailable.
