Client Application Structures

The following structures are supported for client application development by the Windows Biometric Framework API.

In this section

Topic Description
WINBIO_ANTI_SPOOF_POLICY Represents the antispoofing policy for a user.
WINBIO_ASYNC_RESULT Contains the results of an asynchronous operation.
WINBIO_BDB_ANSI_381_HEADER Specifies information about a series of fingerprint or palm samples.
WINBIO_BDB_ANSI_381_RECORD Contains information about a single fingerprint or palm sample from an end user.
WINBIO_BIR Represents a biometric information record (BIR).
WINBIO_BIR_DATA Specifies the size, in bytes, and the offset of a block of biometric information.
WINBIO_BIR_HEADER Contains the header of a biometric information record (BIR).
WINBIO_BSP_SCHEMA Describes the capabilities of a biometric service provider.
WINBIO_EXTENDED_ENGINE_INFO Contains information about the capabilities and enrollment requirements of the engine adapter for a biometric unit.
WINBIO_EXTENDED_ENROLLMENT_STATUS Contains additional information about the status of an enrollment that is in progress.
WINBIO_EXTENDED_SENSOR_INFO Contains information about the capabilities and enrollment requirements of the sensor adapter for a biometric unit.
WINBIO_EXTENDED_STORAGE_INFO Contains information about the capabilities and enrollment requirements of the storage adapter for a biometric unit.
WINBIO_EVENT Contains status information sent to the callback routine when an event notice is raised.
WINBIO_IDENTITY Contains an identifying value associated with a biometric template.
WINBIO_PRESENCE Contains information about the presence of an individual whose presence is being monitored.
WINBIO_REGISTERED_FORMAT Specifies a registered data format as an owner/format pair.
WINBIO_STORAGE_SCHEMA Describes the capabilities of a biometric storage adapter.
WINBIO_UNIT_SCHEMA Describes the capabilities of a biometric unit.
WINBIO_VERSION Contains the software version number of a biometric service provider component.
