Client Application Enumerations

The following enumeration types are supported by the Windows Biometric Framework API.

In this section

Topic Description
WINBIO_ANTI_SPOOF_POLICY_ACTION Specifies the types of actions you take for the antispoofing policy of a user.
WINBIO_ASYNC_NOTIFICATION_METHOD Defines constants that specify how completion notifications for asynchronous operations are to be delivered to the client application.
WINBIO_CREDENTIAL_FORMAT Defines flags that can be used to specify the end-user credential format.
WINBIO_CREDENTIAL_STATE Defines values that specify whether a credential has been associated with the biometric data for an end user.
WINBIO_CREDENTIAL_TYPE Defines flags that can be used to filter on the credential type.
WINBIO_POLICY_SOURCE Lists the possible sources of policy information for the detection of spoofing for biometric factors.
