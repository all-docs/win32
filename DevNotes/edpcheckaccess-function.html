EdpCheckAccess function

[!NOTE] Starting in July 2022, Microsoft is deprecating Windows Information Protection (WIP) and the APIs that support WIP. Microsoft will continue to support WIP on supported versions of Windows. New versions of Windows won't include new capabilities for WIP, and it won't be supported in future versions of Windows. For more information, see Announcing sunset of Windows Information Protection.

For your data protection needs, Microsoft recommends that you use Microsoft Purview Information Protection and Microsoft Purview Data Loss Prevention. Purview simplifies the configuration set-up and provides an advanced set of capabilities.

In the presence of WIP policy, checks if transfer of data sharing or transfer should be allowed between the provided source and target enterprise IDs.

Syntax

HRESULT
EdpCheckAccess(
    _In_opt_ PCWSTR sourceEnterpriseId,
    _In_opt_ PCWSTR targetEnterpriseId,
    _Out_    EDP_POLICY_RESULT* result
    );

Parameters

sourceEnterpriseId

Enterprise ID of the source of the action.

targetEnterpriseId

Enterprise ID of the target of the action.

result

Receives a pointer to a member of the EDP_POLICY_RESULT enumeration.

Return value

An HRESULT value.

Remarks

Enterprise IDs can be NULL to indicate personal context.
