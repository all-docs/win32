Remote Control Persistent Virtual Channels

On WindowsÂ XP, a DLL can specify one or more virtual channels as remote control persistent. A remote control persistent virtual channel is not closed when a remote control connects or disconnects for the session connected to the client. Data will continue to be transmitted and received through this channel. Virtual channels that the DLL does not specify as remote control persistent will close in this scenario.

Remote control persistent virtual channels are specified during the call to VirtualChannelInit. Refer to the reference page for VirtualChannelInit for specific information about this.
