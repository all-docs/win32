What's New in Windows Server 2008 R2

Windows Server 2008 R2 introduces the following new programming elements for Remote Desktop Services.

Element Description
Terminal Services Is Now Remote Desktop Services Terminal Services has been renamed Remote Desktop Services. A server that has the Remote Desktop Session Host (RD Session Host) role service installed is now called an RD Session Host server, instead of a terminal server.Terminal Services Is Now Remote Desktop Services
Remote Desktop Virtualization API The Remote Desktop Virtualization API provides enumerations, interfaces, and structures that you can use to create custom plug-ins that override the standard redirection logic of Remote Desktop Connection Broker (RD Connection Broker). RD Connection Broker (formerly known as TS Session Broker) was expanded to support connections to virtual machines.Using the Remote Desktop Virtualization APIRemote Desktop Virtualization API Reference
Remote Desktop Protocol Provider The Remote Desktop Protocol Provider API can be used to create a protocol that customizes the interaction between the Remote Desktop Services service and desktop clients.Creating a Remote Desktop Protocol ProviderRemote Desktop Protocol Provider Reference
Remote Desktop Services AudioEndpoint API The Remote Desktop Services AudioEndpoint API supports enumeration, interfaces, and structures for audio endpoint registration and data transport.Remote Desktop Services AudioEndpoint API Reference
RemoteApp and Desktop Connection Management Service API The RemoteApp and Desktop Connection Management Service API supports interfaces and structures that you can use to perform custom filtering of resources and provide support for file types that are not natively supported in Windows Server 2008 R2.RemoteApp and Desktop Connection Management Service API Reference
