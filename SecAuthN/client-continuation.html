Client Continuation

Some protocols require multiple authentication messages. In this case, the client parses the response from the server and calls InitializeSecurityContext (General) again using the continue status from the previous call. The client checks the return status from this call and might be required to continue for additional legs. It uses the information in pOutput to construct a message and sends it to the server.
