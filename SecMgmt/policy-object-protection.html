Policy Object Protection

The Policy object is protected by default with the following settings:

  • The local group WORLD has GENERIC_EXECUTE access.
  • The well-known ID System is granted POLICY_ALL_ACCESS.
  • The local group LOCAL_ADMIN has GENERIC_READ, GENERIC_WRITE, and GENERIC_EXECUTE access.
  • The group LOCAL_ADMIN is assigned as owner and primary group of this object.

The Policy object cannot be created or destroyed.
